## ✅ 구현된 기능

### 1. 데모 로그인 버튼
- **Admin 로그인** (파란색) - 관리자 계정
- **User1 로그인** (초록색) - 일반 사용자 1  
- **User2 로그인** (보라색) - 일반 사용자 2

### 2. Mock 데이터 시스템
- 각 사용자별 독립적인 할일 목록
- 카테고리 데이터
- 캘린더 요약 데이터
- 완료 기록 시뮬레이션

### 3. API 차단 시스템
- 데모 토큰 감지 시 실제 서버 API 호출 차단
- 401 에러 방지
- 조용한 에러 처리

## 🎭 데모 사용자 데이터

### Admin User
- **할일**: 관리자 회의 준비, 시스템 점검 (매일)
- **카테고리**: 업무

### User1
- **할일**: 운동하기 (월수금), 영어 공부
- **카테고리**: 건강, 공부

### User2  
- **할일**: 독서하기 (매일), 쇼핑하기
- **카테고리**: 취미, 개인

## 🔧 기술적 구현

### 파일 수정 목록
- `client/src/screens/LoginScreen.js` - 데모 로그인 버튼 추가
- `client/src/api/axios.js` - 데모 모드 API 차단
- `client/src/api/mockData.js` - Mock 데이터 정의
- `client/src/api/todos.js` - 데모 모드 지원 API 래퍼
- `client/src/api/categories.js` - 카테고리 API 데모 모드 지원
- `client/src/hooks/useTimeZone.js` - 데모 모드에서 서버 호출 스킵
- `client/src/hooks/queries/useCalendarSummary.js` - 캘린더 요약 데모 지원

### 동작 방식
1. 데모 로그인 시 `demo_token_` 접두사가 붙은 토큰 생성
2. API 인터셉터에서 데모 토큰 감지
3. 실제 서버 호출 대신 Mock 데이터 반환
4. 사용자별 독립적인 데이터 관리

## 🚀 사용 방법

1. 앱 실행
2. 로그인 화면에서 원하는 데모 계정 버튼 클릭
3. 각 사용자의 할일 목록 확인
4. 할일 추가/수정/삭제 테스트
5. 카테고리 관리 테스트

## 📝 주의사항

- 데모 모드에서는 데이터가 메모리에만 저장됨 (새로고침 시 초기화)
- 실제 서버 연결 시에는 정상적인 API 호출로 동작
- Google Auth 기능은 별도로 비활성화됨
