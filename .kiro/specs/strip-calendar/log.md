## Update Note (2026-02-16): Monthly -> Weekly 전환 정책 반영

### 적용 목적

1. `weeklyTargetWeekStart` stale 값이 `monthlyTopWeekStart`를 덮어쓰는 문제 제거
2. Monthly -> Weekly 전환 시 `currentDate`가 월간 5주 viewport 안에 보이면 `currentWeekStart` 우선 적용

### 반영 파일

1. `client/src/features/strip-calendar/ui/StripCalendarShell.js`
2. `client/src/features/strip-calendar/hooks/useStripCalendarController.js`
3. `client/src/features/strip-calendar/utils/stripCalendarDateUtils.js`

### 핵심 변경사항

1. `onMonthlySettled`에서 `setWeeklyTargetWeekStart(null)` 추가
2. `Monthly_Mode -> Weekly_Mode` 토글 직전에 `setWeeklyTargetWeekStart(null)` 추가
3. `handleToggleMode:toWeekly`에서 `nextWeek` 결정 규칙 추가
   - `currentDate`가 monthly viewport(5주) 내에 있으면 `currentWeekStart`
   - 아니면 `monthlyTopWeekStart`
4. date util 일반화
   - `isDateVisibleInMonthlyViewport(topWeekStart, dateYmd)` 추가
   - `isTodayVisibleInMonthlyViewport`는 래퍼 유지

### 후속 로그 확인 포인트

1. `useStripCalendarController / action:handleToggleMode:toWeekly`
   - `baseTopWeekStart`, `currentWeekStart`, `isCurrentDateVisibleFromTop`, `nextWeek`
2. `StripCalendarShell / state:snapshot`
   - `mode=weekly` 전환 직후 `weeklyTargetWeekStart` 값 확인
3. `WeeklyStripList` 마운트 시 타겟
   - `targetWeekStart`가 `nextWeek`와 일치하는지 확인
